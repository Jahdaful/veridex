## 1. NATURE OF ANALYSIS

VERIDEX Forensic AI uses artificial intelligence (Claude, developed by Anthropic) to perform probabilistic forensic analysis of digital media. All outputs — including verdicts, manipulation scores, evidence grades, and findings — are **AI-generated assessments**, not the certified opinion of a qualified human forensic examiner.

---

## 2. LIMITATIONS OF AI-BASED FORENSIC ANALYSIS

### 2.1 Analysis Basis
- For image files ≤5MB: analysis is based on visual inspection by an AI vision model plus metadata examination.
- For all other files (video, audio, large images): analysis is based **solely on file metadata, filename patterns, and MIME type** — no actual content analysis is performed.
- The System does **not** perform binary-level signal processing, Fast Fourier Transform (FFT) analysis, facial landmark detection, mel-spectrogram analysis, or other laboratory-grade forensic methods.

### 2.2 Known Limitations
- **False positives:** The System may classify authentic media as manipulated.
- **False negatives:** The System may fail to detect deepfake or AI-generated content.
- **Metadata reliance:** Analysis quality is directly dependent on the integrity of file metadata, which can be fabricated.
- **Novel techniques:** The System may not detect manipulation methods developed after its training data cutoff.
- **Context blindness:** The System has no knowledge of the investigation context, prior intelligence, or corroborating evidence.

### 2.3 Evidence Grade Limitations
Evidence grades (A–D) reflect the confidence and completeness of the AI analysis, not the legal admissibility or probative value of the output.

| Grade | Meaning |
|---|---|
| A | High-confidence analysis — image with full EXIF and vision analysis |
| B | Moderate confidence — partial metadata or no vision analysis |
| C | Low confidence — metadata-only or limited signals |
| D | Minimal confidence — filename and MIME type analysis only |

## 3. NOT A SUBSTITUTE FOR CERTIFIED EXPERT ANALYSIS

**VERIDEX outputs must not be used as the sole or primary basis for:**
- Arrest, search, or detention decisions
- Charging decisions or prosecutorial recommendations
- Court submissions as expert evidence without independent verification
- Identification of individuals as suspects based solely on AI analysis

**Before any System output is relied upon in legal proceedings**, it must be reviewed and independently verified by a qualified, certified digital forensics professional. The reviewing expert should:
1. Conduct independent binary-level analysis of the original evidence file
2. Apply recognized forensic methodologies (e.g., EnCase, FTK, Autopsy)
3. Produce their own certified forensic report
4. Testify to their own findings and methodology, referencing System output as one input only

## 4. CHAIN OF CUSTODY

Use of this System does not substitute for proper evidence handling. Authorized Users must:
- Preserve original evidence files unchanged, using hardware write-blockers where required
- Document the SHA-256 hash of the original file before and after analysis
- Record the use of AI-assisted analysis in the case file
- Note the analysis date, time, and System version in all reports
